# systematic-debugging

用于复杂 bug、偶发失败、根因不明的问题排查。

## 适用场景

- 现象很多，但根因不清楚
- 本地和 CI 表现不一致
- 偶发失败、flaky test
- 需要先复现再决定怎么修

## 标准流程

1. 记录现象和已知证据。
2. 优先复现问题。
3. 缩小范围：按层、按模块、按输入、按环境排查。
4. 建立假设并逐个验证。
5. 区分“症状”和“根因”。
6. 给出最小修复路径。

## 推荐搭配

- 真正修复：进入当前项目的实现工作流
- 如果主要在流水线暴露：`fix-ci`
- 如果是 UI 现象：`browser-verification`
