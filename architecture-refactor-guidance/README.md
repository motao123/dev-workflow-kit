# architecture-refactor-guidance

用于结构性改动、模块边界调整、耦合拆解和安全重构分步设计。

## 适用场景

- 不是功能不清，而是结构不对
- 模块太大、边界混乱、职责泄漏
- 想拆接口、切层、分阶段重构
- 需要先判断怎么改最安全

## 标准流程

1. 先理解当前结构和边界。
2. 找出耦合点、泄漏点、风险点。
3. 识别可以安全切开的 seam。
4. 设计分阶段重构顺序。
5. 明确如何证明行为不变。
6. 给出下一步实现路径。

## 推荐搭配

- 真正落地实现：进入当前项目的实现工作流
- 范围本身不清：`planning`
- 合并前总检查：`ship-readiness`

## 调用示例

```text
用 architecture-refactor-guidance 先判断这个模块拆分应该怎么切边界、怎么分步落地。
```
