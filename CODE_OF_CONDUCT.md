# Code of Conduct

## Our goal

`dev-workflow-kit` 希望成为一个**清晰、实用、可讨论**的协作空间。

无论你是在提新 skill、报 bug、改文档，还是讨论 workflow 边界，都欢迎在尊重彼此的前提下参与。

## Expected behavior

请尽量做到：

- 针对问题本身，而不是针对人
- 用具体例子和上下文说话
- 讨论 skill 边界时保持建设性
- 接受不同的实现思路，但要求它们边界清晰、可验证
- 对文档、示例、eval 的修正保持耐心和善意

## Unacceptable behavior

以下行为不被接受：

- 人身攻击、嘲讽、羞辱、恶意挑衅
- 持续性打断、骚扰或故意制造对立
- 明知无关却反复刷屏
- 带有歧视性、侮辱性或敌意的言论
- 在 issue / PR 中故意制造低质量冲突

## Scope

本行为准则适用于：

- Issues
- Pull requests
- Review comments
- 讨论与反馈场景

## Enforcement

如果你遇到不合适的互动，请联系仓库维护者。

维护者可以根据情况：
- 提醒并要求调整表达
- 关闭不合适的讨论
- 拒绝继续处理带有攻击性或破坏性的互动

## Practical principle

对于这个仓库，最重要的一条原则是：

> 让讨论帮助 skill 变得更清晰，而不是让讨论本身变成阻塞。
