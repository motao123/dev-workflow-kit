# Support

## Where to ask what

为了让仓库保持清晰，建议按问题类型选择入口：

### 1. Bug 或行为不一致
请使用：`Bug report`

适合：
- skill 行为不符合预期
- router / eval / examples 不一致
- 文档与实际 skill 边界不一致

### 2. 文档改进
请使用：`Docs improvement`

适合：
- README 不清楚
- examples 不够好
- catalog / design notes 需要更新

### 3. 新 skill 想法
请使用：`Skill proposal`

适合：
- 想新增 skill
- 想扩展某个 workflow 能力方向
- 想先讨论 skill 边界是否独立成立

## Before opening an issue

建议先看：
- `README.md`
- `CONTRIBUTING.md`
- `.github/ISSUE_TEMPLATE/` 下的对应模板

## Security-related concerns

如果问题涉及敏感安全内容，请不要公开发 issue。
请查看：
- `SECURITY.md`

## What this repo is best at

这个仓库最适合讨论：
- workflow skills 的边界
- 交付流程中的瓶颈
- 文档、examples、eval 的一致性
- skill catalog 的组织方式

## What this repo is not for

这个仓库不适合：
- 一般编程问答论坛式提问
- 与 skill 仓库无关的产品支持
- 没有上下文的宽泛抱怨

## Good support requests are concrete

更容易被处理的问题通常会包含：
- 具体 skill 或文件名
- 真实 prompt / 输入示例
- 期望行为与实际行为
- 为什么这个问题会影响使用
