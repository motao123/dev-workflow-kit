# planning

用于在真正动手写代码前，把模糊任务变成可执行计划。

## 适用场景

- 需求还不清楚
- 改动范围较大
- 涉及多模块、多层次
- 需要先定义验收标准
- 不确定验证怎么做最合适

## 标准流程

1. 先读相关代码、文档、约束。
2. 明确目标和不做什么。
3. 把任务拆成最小可执行切片。
4. 识别风险、依赖、前置条件。
5. 设计验证方案。
6. 给出推荐实施顺序。

## 推荐搭配

- 实现阶段：`trellis-lightweight-tdd`
- 执行中卡住：`systematic-debugging`
- 完成前检查：`ship-readiness`

## 调用示例

```text
用 planning 先把这个需求拆成具体实施步骤和验收标准。
```

```text
用 planning 先判断这个重构应该怎么切分，避免一次动太多。
```
