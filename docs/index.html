What it is

dev-workflow-kit is a standalone, markdown-first collection of software-delivery workflow skills. It is not a code generator and not a writing-style guide — it is a router that helps Claude Code / Codex / compatible agents choose the right delivery skill for the task in front of them.

面向 Claude Code / Codex / 兼容 Agent，把开发流程中真正反复出现的环节拆成可调用的独立 skill。

  • Engineers using AI agents through real delivery workflows, not just code completion.
  • Teams that want explicit boundaries for planning, debugging, review, security, rollout, and post-incident work.
  • Operators who treat skill packs as living docs, with evals and prompt examples.

21 skills, grouped by lifecycle

Each skill is its own folder on GitHub with SKILL.md, README, evals, and prompt examples.

Plan & scope

4 skills
planning Turn fuzzy requests into a concrete implementation plan with risks and acceptance criteria. test-strategy-gap-review Decide whether current verification is sufficient and what coverage is missing. change-impact-analysis Map blast radius across modules, clients, configs, and verification surfaces. architecture-refactor-guidance Choose safe boundaries and refactor sequencing before implementation.

Diagnose

4 skills
systematic-debugging Reproduce, narrow, and root-cause unclear bugs before changing code. incident-observability-triage Triage live incidents starting from alerts, logs, metrics, or traces. performance-investigation Narrow latency, throughput, or resource regressions before optimizing. fix-ci Read the earliest failure, classify, and produce a minimal repair path.

Pre-merge control

7 skills
review-address-comments Group, classify, and resolve PR review comments with a clear summary. browser-verification Smoke-test UI behavior with explicit observable evidence. security-review Identify trust-boundary, abuse-case, and unsafe-default risks. dependency-compliance-audit Review third-party package risk, license, and supply-chain fit. api-contract-compatibility-review Check breaking changes, versioning, and migration guidance for consumers. cost-impact-review Estimate infra and per-request cost regressions before merge. data-quality-audit Audit runtime data correctness, freshness, completeness, and integrity.

Release & delivery

6 skills
release-rollout-strategy Design phased, canary, or flag-gated rollout with checkpoints and rollback rules. environment-config-audit Catch config drift, env mismatch, and deploy-time assumption risk. data-migration-safety-review Review schema migrations, backfills, and coexistence windows for safety. release-incident-postmortem Run a structured retrospective after an incident has been mitigated. ship-readiness Final gate: blockers, verification status, and risk summary before merge or release. docs-writer Produce README, migration notes, release notes, and usage docs grounded in real code.

Quick Start

Drop the repository into your skills directory and call any skill by name.

Install path

~/.codex/skills/dev-workflow-kit

Or clone directly

git clone https://github.com/motao123/dev-workflow-kit ~/.codex/skills/dev-workflow-kit

Pick by current bottleneck, not the final goal

  • Stuck on "what should we build?" → planning
  • Stuck on "what does this change touch?" → change-impact-analysis
  • Stuck on "why is this failing?" → systematic-debugging
  • Stuck on "is this ready to ship?" → ship-readiness

First call

用 dev-workflow-kit 的 planning 先把这个需求拆成实现步骤、风险和验证方案。
