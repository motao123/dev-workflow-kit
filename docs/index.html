是什么

不是写代码助手，是
交付流程的导航器

dev-workflow-kit 是一个独立的、markdown-first 的软件交付工作流 skill 集合，专为 Claude Code / Codex / 兼容 Agent 设计。

它不是代码生成器，也不是写作风格指南——它是一个路由器，帮 AI Agent 在每一个真实的交付环节里找到对的 skill。

谁在用
  • 把 AI Agent 推进真实交付流程的工程师，而不只是用它做代码补全
  • 希望对规划、调试、评审、安全、发布、复盘有明确边界的团队
  • 把 skill pack 当作可演进文档来运营，配合 evals 和 prompt 示例
skills

21 个 skill，按交付阶段分组

每个 skill 都是 GitHub 上独立的目录，包含 SKILL.md、README、evals 和 prompt 示例。

阶段 1

规划与拆解

4 skills
  • planning把模糊需求变成实施步骤、风险和验收标准。
  • test-strategy-gap-review评估测试是否够用、缺什么覆盖、最该补哪一类。
  • change-impact-analysis分析改动影响哪些模块、消费者、配置和验证面。
  • architecture-refactor-guidance判断结构调整和重构的安全切边界与分步顺序。
阶段 2

诊断与排障

4 skills
  • systematic-debugging复现、缩小范围、定位根因，再决定怎么修。
  • incident-observability-triage基于告警、日志、指标、链路做线上事故分诊。
  • performance-investigation在优化之前先定位延迟、吞吐、资源瓶颈。
  • fix-ci读最早失败点，分类，给出最小修复路径。
阶段 3

合并前控制

7 skills
  • review-address-comments把 PR 评论分组、分类、逐条处理并给出解决摘要。
  • browser-verification浏览器级 UI smoke test，给出可观察的证据。
  • security-review识别信任边界、滥用面、不安全默认值。
  • dependency-compliance-audit第三方依赖、license、合规、供应链评估。
  • api-contract-compatibility-review检查 API/schema 的兼容性、versioning 与迁移建议。
  • cost-impact-review评估改动是否会让 infra 或单请求成本明显上涨。
  • data-quality-audit运行期数据正确性、新鲜度、完整性、约束完整审查。
阶段 4

发布与交付

6 skills
  • release-rollout-strategy设计灰度、canary、flag-gated 上线与回滚条件。
  • environment-config-audit审计环境一致性、配置漂移、flags、secrets。
  • data-migration-safety-review数据迁移、backfill、并存窗口的安全审查。
  • release-incident-postmortem事故缓解后的结构化复盘与改进项闭环。
  • ship-readiness合并/发布前的最终检查、blocker、风险摘要。
  • docs-writer基于真实代码起草 README、迁移说明、release note。
quick start

三步开始用

把仓库放进 skill 目录，按当前瓶颈调用对应的 skill。

1 · 安装路径

~/.codex/skills/dev-workflow-kit

2 · 直接 clone

git clone https://github.com/motao123/dev-workflow-kit \
  ~/.codex/skills/dev-workflow-kit

3 · 第一次调用

用 dev-workflow-kit 的 planning 先把这个需求
拆成实现步骤、风险和验证方案。

按当前瓶颈选 skill

  • 需求没拆清 → planning
  • 不知道这次改动会影响谁 → change-impact-analysis
  • 不知道为什么坏了 → systematic-debugging
  • CI 不过 → fix-ci
  • 能不能上线 → ship-readiness
  • 事故已缓解但要复盘 → release-incident-postmortem
