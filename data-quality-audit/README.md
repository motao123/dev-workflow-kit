# data-quality-audit

用于审查运行期数据的正确性、完整性、新鲜度、一致性。

## 适用场景

- 某张表 / 数据集 / 事件流取值不对
- 怀疑有空值、重复、外键或枚举非法
- 怀疑数据不新鲜或缺失
- 下游报表或行为异常但代码路径看起来没问题
- 发版或开放新接口前想做一次数据可信度检查

## 标准流程

1. 确认要审查的数据集和它的用途。
2. 列出相关质量维度：正确性、完整性、新鲜度、唯一性、引用完整性、schema 合规。
3. 找出异常和可能的失效模式。
4. 区分关键问题和低优先级清理。
5. 给出修复和验证建议。
6. 标出后续需要的监控或合约。

## 推荐搭配

- 修复要动 schema / backfill：`data-migration-safety-review`
- 看起来是代码导致的：`systematic-debugging`
- 需要写数据合约 / 文档：`docs-writer`

## 调用示例

```text
用 data-quality-audit 评估这个表的新鲜度、完整性和重复行情况。
```
