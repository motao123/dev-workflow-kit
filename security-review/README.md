# security-review

用于在高风险改动上做一次安全视角审查。

## 适用场景

- 鉴权、权限、角色逻辑变更
- 用户输入进入数据库、命令、模板、文件系统
- 涉及 token、secret、回调、上传、重定向
- merge 前想补一次安全检查

## 标准流程

1. 识别资产、边界、用户输入。
2. 找出攻击面。
3. 检查注入、越权、泄漏、不安全默认值等常见风险。
4. 判断优先级或严重度。
5. 给出最小有效缓解方案。
6. 给出验证建议。

## 推荐搭配

- 真正改代码：进入当前项目的实现工作流
- 合并前综合判断：`ship-readiness`
- 需要补文档：`docs-writer`

## 调用示例

```text
用 security-review 看一下这个鉴权改动，列出主要风险和修复建议。
```
